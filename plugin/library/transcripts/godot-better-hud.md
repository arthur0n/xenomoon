**Source** ‚Äî `godot-better-hud.md` (raw now in `transcripts/archive/godot-better-hud.md`). Solo-dev tutorial: "the most common Godot UI container nodes and how to use them" (hexagod-style). No URL captured.
**Why harvested** ‚Äî about to build "Better HUD": today's HUD is a bare `CanvasLayer` + manually-offset `Label`s (`scripts/objective_hud.tscn` ‚Äî anchors + hand-typed `offset_*`, no containers). **Points**

| # | Point (technique/claim) | Valid for our stack? | Already learned? | Where / gap | Verdict |
|---|---|---|---|---|---|
| 1 | Core thesis: nest containers recursively (containers in containers, display elements as leaves) ‚Üí engine auto-places children ‚Üí responsive layouts that adapt to screen size. | 4 | `GridContainer`: auto-grid; `columns` sets wrap count; independent h/v separation; per-child `expand` on either axis; row/col sized to largest cell. | holds | **gap** | Not in basic-hud (which only did HBox/VBox/Margin/Panel).
