# Stylized Grass Shaders in Godot — transcript digest

**Source** — `godot-shader-pixel-art.md` (the raw, now in `transcripts/archive/godot-shader-pixel-art.md`) (Godot 4.3 stylized-grass devlog: multimesh quads, billboarding, wind, fake-perspective under orthographic, low-frame-rate quantization, player/character displacement, hybrid toon shading, cloud-shadow noise via global shader + include file).

**Why harvested** — the harvest brief named **Phase 6 outline shaders** (`godot-edges-depth`, `godot-edges-normal`, `godot-outline-compositing`; design `design/post-process-outlines.md`) as the build prompting this. **Mismatch flagged honestly:** this transcript is about _world-space material/vertex shading of grass_, not screen-space post-process edge detection. It shares almost nothing with the outline build. The mapping below is therefore mostly "out of scope / Later"; the one genuinely useful cross-over point for our pixel-art pipeline is the orthographic-camera shader caveat (#3) and the hybrid toon-shading idea (#8), neither of which is needed for Phase 6.

**Stack reminder for the mapping** — Godot 4.3+, Forward+, reversed-Z, orthographic fixed-angle camera (skill: godot-camera-rig), 3D pixel art rendered in a SubViewport (nearest filter, AA off; skill: godot-3d-pixelation), GDScript-only, composition over inheritance. Current build is a graybox blockout heading into screen-space outlines — no foliage, no weather, no vegetation in the roadmap (Phases 6 outlines, 7 assets, 8 characters/animation).
