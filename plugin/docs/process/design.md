# Design docs

Each file here is one small, agreed slice of work produced by **the orchestrator** (Opus). A doc must be implementable by the active domain's builder in one task and verifiable by the domain's checks.

Workflow: idea → orchestrator (interviews, cuts scope, writes doc) → builder (implements) → verify (checks) → human look.

No code is designed or written here beyond the agreed slice — parked ideas live in each doc's "Later" section.
