# tools/CAPABILITIES.md — registered framework tools

| Tool                    | What it does                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     | Invocation                                                                                                                                                                                                                                                                                                                                                       | Notes                                                                                                                                                                                                                                                                      |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `validate.sh`           | Format + lint + parse + scene property check + smoke run (godot-verify layers 1–2)                                                                                                                                                                                                                                                                                                                                                                                                               | `tools/validate.sh [scene.tscn]`                                                                                                                                                                                                                                                                                                                                 | Requires `--headless`; stops at first failure; exit 0 = OK                                                                                                                                                                                                                 |
| `playgrade.sh`          | The evaluator's deterministic grader (the deterministic half of the generator–evaluator loop) — composes `tools/lib/checks.sh` + the game's adversarial `play_*.gd` bots into a 5-criterion rubric (runs-clean / renders-healthy / core-loop-functional / data-driven-adherence / feel-responsive) and writes a structured `.xenodot/playgrade/<slug>.json` (per-criterion PASS/FAIL/SKIP + aggregate exit code: 0 iff overall PASS; SKIPs never fail). Invoked by the `godot-playtester` agent. | `tools/playgrade.sh --slug <slug> [--design design/<slug>.md] [--scene <scene.tscn>] [--out <path>]` — writes `.xenodot/playgrade/<slug>.json`.                                                                                                                                                                                                                  | Needs the engine (`$GODOT`). Runs headless: runs-clean + core-loop graded; render/data/feel SKIP in v1 (render needs a display → godot-verify L3). Rubric contract: skill `godot-playgrade`.                                                                               |
| `smoke_scene_errors.sh` | Per-scene headless error capture (godot-verify layer 2b), runnable standalone — fails on any non-benign ERROR / name-clash line from a headless scene instantiate. Thin wrapper over `check_scene_errors` (`tools/lib/checks.sh`), which `validate.sh` + `playgrade.sh` call directly.                                                                                                                                                                                                           | `tools/smoke_scene_errors.sh`                                                                                                                                                                                                                                                                                                                                    | Headless OK.                                                                                                                                                                                                                                                               |
| `verify_scene.gd`       | Scene property validation — detects silently-dropped unknown properties (godot-verify layer 1)                                                                                                                                                                                                                                                                                                                                                                                                   | `$GODOT --headless --path . --script tools/verify_scene.gd [-- scene.tscn]`                                                                                                                                                                                                                                                                                      | Headless OK                                                                                                                                                                                                                                                                |
| `verify_render.gd`      | Render check — fails if output is flat color; saves `.godot/verify_render_last.png` (godot-verify layer 3)                                                                                                                                                                                                                                                                                                                                                                                       | `$GODOT --path . --resolution 640x360 -s tools/verify_render.gd [-- scene.tscn]`                                                                                                                                                                                                                                                                                 | Requires display (NO `--headless`); window flashes briefly                                                                                                                                                                                                                 |
| `capture_screenshot.gd` | Renders a scene via Main's SubViewport (640x360 render rig) and saves a PNG for visual inspection of geometry, scale, and placement. Supports standard `cam_pos` teleport and **diagnostic mode** (inject a temporary top-down orthographic camera to see inside enclosed rooms).                                                                                                                                                                                                                | `$GODOT --path . --resolution 640x360 -s tools/capture_screenshot.gd [-- <scene> [out.png [cam_pos [diag_look_at [diag_ortho_size]]]]]` — `cam_pos` = `"X,Y,Z"` world position; `diag_look_at` = `"X,Y,Z"` target (activates diagnostic mode — temporary Camera3D, CameraRig disabled for capture); `diag_ortho_size` = ortho size in world units (default 8.0). | Requires display (NO `--headless`). Diagnostic camera is removed after capture; scene file is never modified. For walled rooms prefer a low interior vantage (eye ~6 m up, look at room centre, ortho ~8) over a steep top-down — see `library/tools/scene-screenshot.md`. |
| `gen_textures.gd`       | Procedurally generates placeholder textures for materials                                                                                                                                                                                                                                                                                                                                                                                                                                        | `$GODOT --headless --path . --script tools/gen_textures.gd`                                                                                                                                                                                                                                                                                                      | Headless OK                                                                                                                                                                                                                                                                |
| `gen_models.gd`         | Procedurally generates placeholder low-poly `.glb` prop models (primitive kitbash → GLTFDocument)                                                                                                                                                                                                                                                                                                                                                                                                | `$GODOT --headless --path . --script tools/gen_models.gd`                                                                                                                                                                                                                                                                                                        | Headless OK; placeholder art (skill `godot-procedural-model`)                                                                                                                                                                                                              |
| `install_gdstyle.sh`    | Installs the gdstyle Godot editor plugin (pinned v0.1.7) into the game's `addons/` — the in-editor **advisory** GDScript linter (config `gdstyle.toml`). The blocking gate stays gdformat + gdlint (`validate.sh`).                                                                                                                                                                                                                                                                              | `tools/install_gdstyle.sh` (run from the project root)                                                                                                                                                                                                                                                                                                           | Needs network; not headless. Rationale + limits: `library/tools/gdscript-linter.md`                                                                                                                                                                                        |
| `forge-facts`           | Print one deterministic project fact from the generated manifest (`.xenodot/manifest.json`) — engine bin/version, render config, build/verify commands, input actions, layout, tool registry — so an agent reads a value instead of re-deriving it (re-reading project.godot, re-globbing `tools/`).                                                                                                                                                                                             | `tools/forge-facts [dotted.key]` — e.g. `tools/forge-facts commands.validate`, `tools/forge-facts render.renderer`, `tools/forge-facts capabilities`; no arg prints the whole manifest.                                                                                                                                                                          | No engine needed; reads the manifest the framework regenerates on startup/`doctor`/materialize. `$GODOT` is already exported, so you don't need this just to run the engine.                                                                                               |

## Runtime library (`tools/lib/`)

`class_name` static-util modules the GAME preloads at runtime — reference by bare class name (no
`preload` path). Copied with the other tools and held to the same strict gate (`validate.sh` lints
`tools/`). Add one when a 3rd near-duplicate appears (see `godot-code-rules` DRY rule).

| Module                        | `class_name`   | What it gives                                                                                                                                                                             |
| ----------------------------- | -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tools/lib/node_builder.gd`   | `NodeBuilder`  | Build owned scene nodes — `box_body` / `ramp_body` / `trigger` / `vis_box` / `attach` — without the new/name/add_child/set-owner boilerplate.                                             |
| `tools/lib/mesh_flasher.gd`   | `MeshFlasher`  | `flash(host, mesh_root, color, dur)` — parallel albedo+emission flash over all MeshInstance3D parts (unique material each); returns the Tween to chain `queue_free`/restore.              |
| `tools/lib/audio_one_shot.gd` | `AudioOneShot` | `play_detached(player, host)` — reparent a one-shot AudioStreamPlayer to a surviving node so `queue_free()` doesn't cut the tail, then auto-free (the godot-audio "despawn SFX" pattern). |
| `tools/lib/players.gd`        | `Players`      | `current(tree)` / `reset_to(player, pos, rot_y)` — single home for the player group-lookup + teleport-reset.                                                                              |
