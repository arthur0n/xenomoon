## Sections (segmented)

The video has two halves; the second (graphics) splits into the "3 pillars". For the asset push the load-bearing sections are S2–S5.

- **S1 — Functionality / prototyping** (node 2D→3D parity, units=metres, CSG greybox, character controller, Jolt physics).
- **S2 — Asset terminology + formats** (mesh/UV/texture/material/rig/anim; glTF/glb/.blend; re-export via Blender).
- **S3 — Import + materials** (glTF-as-scene, Advanced Import: per-node skip / material override-or-extract / embedded textures; StandardMaterial3D PBR maps albedo/metallic/roughness/normal/AO/emission; ORM channel-pack; normal-map invert-Y).
- **S4 — Using the model in-game: scene structure + collision** (inherited vs **nested** vs make-local; simple-shape vs multi-shape vs auto-generate collision; Make-Unique on duplicated CollisionShape; name suffixes `-noimp`/`-col`/`-colonly`/`-loop`; import scripts).
- **S5 — Greybox→asset MIGRATION** (replace CSG/box greybox with shared-material building assets, lock/hide greybox while swapping, decorate after walls land).
- **S6 — Animated characters** (skeleton+meshes+AnimationPlayer from glTF; separate model/anim glTFs → AnimationLibrary; retargeting Mixamo).
- **S7 — Lighting + environment** (WorldEnvironment background drives ambient; 3 light types; tonemap Filmic/ACES/AgX; LUT/GradientTexture1D; camera attributes/auto-exposure; render quality knobs).

## Recommended next — per section, each routable individually

Bucket 3/4 (no-brainers / adopt-with-rework) from the 6-bucket frame; one line each.

- **S3+S4 — glTF Advanced-Import workflow + scene-structure + collision (#7, #10, #11) → skill-researcher.** Single highest-value gap for the asset push. EXTEND `godot-mesh-import-pixel-art` (or evaluate a new `godot-mesh-import` HD sibling): Advanced-Import per-node skip, material override-vs-extract, embedded-image handling; the inherited-vs-**nested**-vs-make-local decision (nested = default, separates logic from art = our composition principle); auto-generate-collision-from-mesh + Make-Unique on duplicated CollisionShape. Without this, builders swapping greybox→glb have no canonical workflow.
- **S3 — HD PBR material import (#8, #9) → skill-researcher (NEW skill) AND asset-advisor/art-director (decision).** This is the convention conflict: HD wants linear filter + mipmaps + StandardMaterial3D PBR (albedo/metallic/roughness/normal/AO/emission, ORM pack, normal invert-Y), which the NEAREST/no-mipmap `*-import-pixel-art` skills forbid. Route the import-technique gap to skill-researcher (a `godot-material-pbr` / HD-texture-import skill) and the filter/mipmap/PBR-vs-flat _art decision_ to asset-advisor/art-director first so the skill is briefed correctly.
- **S5 — Greybox→asset MIGRATION (#13) → skill-researcher (NEW skill).** Nothing owns the REPLACE half of the loop. `godot-greybox` builds blockouts; we need a `godot-greybox-to-asset` (or extend `godot-greybox` with a "retire the blockout" section): shared-material building set, per-model collision, swap order, lock+transparent-magenta-hide the greybox while swapping, delete greybox last, decorate after walls. This is literally the push's headline task.
- **S1 — Jolt physics engine (#4) → game-designer (project-config decision).** On 4.6 Jolt is built-in; decide whether to set `physics/3d/physics_engine = JoltPhysics3D` and record it in CLAUDE.md. Cheap, one-line, but a real decision — not a skill.
